Overview

This tutorial demonstrates how to create an integration solution that splits a message into multiple messages graphically.

In this tutorial, a PI file is provided with the solution. You will import the solution to your IBM Integration Toolkit workspace, deploy it to your default test environment, and test the solution with the Flow exerciser.

Import the integration solution into the development environment

Click Import to add the integration solution to your IBM Integration Toolkit workspace.

A shared library project and an application project are added in the Application Development view of your IBM Integration Toolkit workspace. These two projects contain all the development and test resources that you need to implement and run this solution.

Find out more

  • Integration Development perspective

Import

Deploy the integration solution into the test environment

In IBM Integration Bus, after you develop your solution, you deploy your integration solution to an integration server within an integration node.

Click Deploy to deliver the solution from the development environment to the default, local test environment.

When you click Deploy, the shared library project and the application project are deployed into the default test environment: TESTNODE_name_of_your_system integration node, default integration server.

You can check the Integration Nodes view in your IBM Integration Toolkit workspace to see the default integration node. If you expand the integration node, you can see the default integration server, as well as deployed resources.

Test the solution

In the Application Development view of your workspace.

  1. Open the SplitMessageIntoMultipleMessages.msgflow, and click the Flow Exerciser icon to start recording the message path through the flow.
  2. Click the Send Message icon to select a message to send to the flow.
  3. Choose the ExampleInputMessage1, edit the message data if you like, and click Send. Your request message is sent to the HTTP input node.
    • Substep one
    • Substep two

After the request message is processed, the message path is automatically highlighted on the message flow.

Click on any connection to see the data that passed through that connection. You can see that the request (input) message data has two fields named InputField1 and InputField2. The response (output) message has converted this to fields named OutputField1 and OutputField2.
